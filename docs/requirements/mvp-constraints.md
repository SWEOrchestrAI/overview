# MVP Constraints

The MVP should avoid over-expanding into a full SaaS platform too early.

Initial constraints:

- no direct cloud AI coding execution,
- no full IDE replacement,
- no automatic destructive Git operations,
- no uncontrolled shell access,
- no provider lock-in,
- no assumption that all users use the same AI assistant,
- no cloud requirement to upload full source code,
- no complex multi-tenant enterprise model before validating the local/cloud workflow.
